This application is used to test the fingertip skin by applying controlled force on it.

Description

In this application the IAI Tabletop robot is used to apply controlled force on an iCub fingertip, or any kind of skin patch, in order to test it. The force is controlled using a force/torque sensor which is used to probe the fingertip.

The typical scenario includes four modules:

  • IAITableTopController
  • NIDAQmxReader
  • robotInterface
  • dataDumper

Module Breakdown

Below is a block diagram showing the module architecture of the Fingertip Testing application.

Fingertip Testing module architecture.

IAITableTopController

During these experiments the robot usually moves over a set of discrete steps described by appropriately configuring the IAITableTopController module. This module which sends position commands and receives status messages via USB serial port. It also constantly polls the robot position and outputs it on a YARP BufferedPort.

NIDAQmxReader

Data from the force/torque sensor is acquired via the NIDAQmxReader module. This module relies upon the National Instruments proprietary data acquisition (NIDAQmx/NIDAQmxBase) library and C API. The data is sampled at into a circular buffer of size from which samples are read at each module iteration.

robotInterface

The fingertip skin sensor is acquired by running a robotInterface module which initialises the CAN bus communication with the fingertip. This module also creates an output BufferedPort on which the sensor data is reversed. It is part of the standard iCub robot setup.

The robotInterface must be configured to load a robot with only the skin devices we wish to test. This is done by appropriately modifying the xml files in icub-main/app/robots/SkinTableTop/.

dataDumper

The data acquired from all the sensors is then dumped to txt files using four instances of the dataDumper module.

How to Run the Application

Customize the file app/applications/*.template for your specific platform before launching it.
